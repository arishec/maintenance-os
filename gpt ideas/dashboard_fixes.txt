==================================================
1. DASHBOARD COUNTS LOGIC (CRITICAL)
==================================================

PROBLEM:
Counts do not align with issue statuses.

Current:
- Open issues: 3
- Awaiting quotes: 2
- One issue shows "Quotes Received"

This creates confusion.

--------------------------------------------------
CORRECT STATE MODEL
--------------------------------------------------

Each issue must be in EXACTLY ONE state:

- awaiting_quotes → no contractor replies
- quotes_received → at least one reply exists
- active_job → contractor selected
- completed
- canceled
- archived

--------------------------------------------------
FIX (BACKEND LOGIC)
--------------------------------------------------

Define derived states:

awaiting_quotes:
- dispatches exist
- NO ContractorResponse records

quotes_received:
- at least 1 ContractorResponse exists

--------------------------------------------------

Dashboard counts:

Open issues:
- all issues where status NOT IN (completed, canceled, archived)

Awaiting quotes:
- issues where status == awaiting_quotes

Quotes received (NEW METRIC — recommended):
- issues where status == quotes_received

Active jobs:
- status == active_job

--------------------------------------------------
IMPORTANT RULE:
Statuses must be mutually exclusive.

DO NOT allow:
- issue counted in both awaiting_quotes and quotes_received

==================================================
2. NOTIFICATIONS (CRITICAL FIX)
==================================================

PROBLEM:
Contractor reply occurred but no notification shown.

This breaks the core product loop.

--------------------------------------------------
REQUIRED BEHAVIOR
--------------------------------------------------

Trigger notification when:

1. First contractor reply arrives
2. Any new contractor reply arrives
3. Contractor asks a follow-up question

--------------------------------------------------
NOTIFICATION STRUCTURE
--------------------------------------------------

Example:

title:
New contractor reply

body:
Test Plumber sent a $450 quote

metadata:
- issueId
- contractorId
- responseId

--------------------------------------------------
DATABASE
--------------------------------------------------

Create Notification record:

- id
- userId
- issueId
- type = "contractor_reply"
- message
- read (boolean)
- createdAt

--------------------------------------------------
UI
--------------------------------------------------

Dashboard notifications panel:
- show latest notifications
- clickable → goes to issue detail
