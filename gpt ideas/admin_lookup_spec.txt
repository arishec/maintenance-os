This document reviews and improves the internal admin lookup endpoint.

Goal:
- Keep it lightweight (no UI)

==================================================
CURRENT APPROACH (WHAT YOU BUILT)
==================================================

Endpoint:
https://ifbids.com/api/admin/lookup?ref=...

Behavior:
- ISS-XXXXXX → returns full issue chain
- MNT-XXXXXX → returns dispatch via reply token

Access:
- restricted to a specific logged-in user

Response:
- raw JSON

--------------------------------------------------

==================================================
REQUIRED FIXES (IMPORTANT)
==================================================

--------------------------------------------------
1. DO NOT LEAVE THIS EXPOSED WITHOUT HARD GUARDS
--------------------------------------------------

Current risk:
Endpoint is publicly accessible path on production domain.

Even with auth, this is not enough.

--------------------------------------------------

Required fix:

Add strict server-side check:

```ts
if (!user || !ADMIN_EMAILS.includes(user.email)) {
  return new Response("Forbidden", { status: 403 })
}
```

--------------------------------------------------

Add SECOND layer (strongly recommended):

```ts
if (req.headers.get("x-admin-secret") !== process.env.ADMIN_SECRET) {
  return new Response("Forbidden", { status: 403 })
}
```

--------------------------------------------------

Result:
- protects against auth bugs
- protects against accidental exposure
- protects against future dev mistakes

==================================================
2. RENAME THE ROUTE (REDUCE SURFACE AREA)
==================================================

Current:
`/api/admin/lookup`

Problem:
- obvious
- guessable
- easily discoverable

--------------------------------------------------

Recommended:

`/api/internal/lookup`

OR

`/api/dev/lookup`

--------------------------------------------------

This is NOT security through obscurity.

It simply reduces unnecessary attention.

==================================================
3. STANDARDIZE REFERENCE TYPES
==================================================

Current:
- ISS- for issue
- MNT- for dispatch

--------------------------------------------------

Required rule:

```ts
if (ref.startsWith("ISS-")) {
  // issue lookup
} else if (ref.startsWith("MNT-")) {
  // dispatch lookup
} else {
  return error
}
```

--------------------------------------------------

Reason:
- avoids ambiguity
- prevents future confusion
- keeps logic deterministic

==================================================
4. ADD CLEAR NOT FOUND RESPONSE
==================================================

Current risk:
- unclear empty or silent failures

--------------------------------------------------

Required response:

```json
{
  "error": "NOT_FOUND",
  "message": "No issue or dispatch found for reference MNT-QYTQ7P"
}
```

--------------------------------------------------

Reason:
- faster debugging
- eliminates guesswork

console.log("ADMIN_LOOKUP", {
