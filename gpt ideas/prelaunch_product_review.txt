==================================================
CORE ASSESSMENT
==================================================

The core loop is working:

issue → AI classification → dispatch → contractor reply → parsing → selection → scheduling

This is a REAL product.

However, several critical gaps will break trust or stall users.

==================================================
TIER 1 — LAUNCH BLOCKERS (FIX FIRST)
==================================================

--------------------------------------------------
1. PHOTO URL EXPIRATION (CRITICAL)
--------------------------------------------------

Problem:
- Supabase signed URLs expire
- Contractor opens email later → image broken → no response

--------------------------------------------------

Solution Option A (FASTEST):

Make images public:

bucket: public
path: /issues/{issueId}/{filename}

--------------------------------------------------

Solution Option B (BETTER):

Proxy endpoint:

/api/images/{id}

Server:
- generate fresh signed URL
- redirect to it

--------------------------------------------------

DO NOT:
- send short-lived signed URLs in emails

--------------------------------------------------

Impact:
This WILL break on real usage if not fixed.

==================================================
2. JOB COMPLETION FLOW (MISSING)
==================================================

Current:
issue → dispatch → quote → select → schedule → STOP

--------------------------------------------------

Fix:

Add button:
"Mark job complete"

--------------------------------------------------

Backend:

job.status = "completed"
issue.status = "completed"

--------------------------------------------------

Optional:
- upload completion photo
- add completion notes

--------------------------------------------------

Impact:
Without this, the product feels unfinished.

==================================================
3. NOTIFICATIONS (CURRENTLY BROKEN)
==================================================

Current:
- notifications saved in DB
- user never sees them

--------------------------------------------------

Fix (minimum):

On contractor reply → send email:

Subject: New quote received

Body:
"Mike Rodriguez quoted $365 for your issue.
View it here: [link]"

--------------------------------------------------

Do NOT overbuild:
- no push notifications yet

--------------------------------------------------

Impact:
Without this, users miss quotes entirely.

==================================================
TIER 2 — USER RETENTION (FIX NEXT)
==================================================

--------------------------------------------------
4. ONBOARDING FLOW (MISSING)
--------------------------------------------------

Current:

- no guidance

--------------------------------------------------

Fix:

--------------------------------------------------

Auto-open "Report Issue"

--------------------------------------------------

==================================================

- price
- availability
- notes
- selection button

--------------------------------------------------

Mike: $365 — Monday
John: $420 — Tomorrow

--------------------------------------------------

Impact:
This is your main value prop.
