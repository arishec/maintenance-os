MAINTENANCE OS
DASHBOARD ↔ MARKETING NAVIGATION IMPLEMENTATION GUIDE

==================================================
PURPOSE
==================================================

This document explains exactly how to fix navigation so users can move between:

- your marketing site (homepage, features, pricing, guides)
AND
- your app (dashboard, issues, properties, etc.)

Right now:
- logged-out experience is good
- logged-in experience is isolated

This needs to be unified.

==================================================
WHY THIS IS IMPORTANT
==================================================

1. UX
Users expect to move between:
- app
- homepage
- features
- pricing

Right now they cannot.

--------------------------------------------------

2. Growth / conversion
Users who are logged in:
- never see pricing again
- never re-evaluate value
- never explore features

This hurts upgrades later.

--------------------------------------------------

3. Product quality perception
A real SaaS product always feels like:
ONE SYSTEM

Not:
- “marketing site”
- “separate tool”

--------------------------------------------------

4. SEO (indirect benefit)
Even though dashboard is noindex:
- internal linking still matters
- crawl structure still matters

==================================================
CURRENT PROBLEM
==================================================

Dashboard navigation only includes:

- Dashboard
- Properties
- Issues
- Contractors
- History
- Settings

There is NO link to:
- homepage (/)
- features (/features)
- pricing (/pricing)
- guides (/guides)

==================================================
CORRECT ARCHITECTURE
==================================================

You should have TWO navigation layers:

1. TOP NAV (marketing navigation)
2. SIDEBAR (app navigation)

--------------------------------------------------

TOP NAV (visible everywhere)
- Logo (→ homepage)
- Features
- How It Works
- Pricing
- Guides

--------------------------------------------------

SIDEBAR (app only)
- Dashboard
- Properties
- Issues
- Contractors
- History
- Settings

==================================================
FINAL DESIGN MODEL
==================================================

Layout structure:

-------------------------------------
| TOP NAV (marketing links)         |
-------------------------------------
| SIDEBAR | MAIN CONTENT           |
|         |                        |
-------------------------------------

==================================================
MINIMUM FIX (REQUIRED)
==================================================

At absolute minimum:

Make the logo clickable:

CODE:

<Link href="/">Maintenance OS</Link>

This alone allows users to go back to homepage.

BUT this is not enough for a polished product.

==================================================
RECOMMENDED FIX (BEST PRACTICE)
==================================================

Add a full top navigation bar inside your dashboard layout.

==================================================
STEP-BY-STEP IMPLEMENTATION
==================================================

STEP 1 — EDIT YOUR DASHBOARD LAYOUT

Find your main dashboard layout file.

Likely:
app/(app)/layout.tsx

--------------------------------------------------

STEP 2 — ADD TOP NAV HEADER

Add this above your existing layout:

```tsx
import Link from 'next/link';

<header className="border-b bg-white px-6 py-3 flex items-center justify-between">
  <div className="flex items-center gap-6">

    {/* LOGO */}
    <Link href="/" className="font-semibold text-lg">
      Maintenance OS
    </Link>

    {/* MARKETING NAV */}
    <nav className="hidden md:flex gap-6 text-sm">
      <Link href="/features">Features</Link>
      <Link href="/how-it-works">How It Works</Link>
      <Link href="/pricing">Pricing</Link>
      <Link href="/guides">Guides</Link>
    </nav>

  </div>

  {/* RIGHT SIDE */}
  <div className="flex items-center gap-4">
    <Link href="/dashboard" className="text-sm font-medium">
      Dashboard
    </Link>
  </div>
</header>
```

--------------------------------------------------

STEP 3 — KEEP YOUR SIDEBAR

Do NOT remove your sidebar.

Your sidebar is your app navigation.

It should stay exactly as it is.

==================================================
OPTIONAL IMPROVEMENTS
==================================================

1. Highlight active page

Example:

If user is on /features:
- make "Features" bold or underlined

--------------------------------------------------

2. Mobile behavior

On mobile:
- hide full nav
- replace with hamburger menu

--------------------------------------------------

3. Add avatar / user menu

Right side can include:
- user icon
- logout
- settings

--------------------------------------------------

4. Add "Back to site" (optional)

If you want something simpler:

```tsx
<Link href="/" className="text-sm text-muted-foreground">
  ← Back to site
</Link>
```

==================================================
WHAT NOT TO DO
==================================================

Do NOT:

- remove the sidebar
- mix app links and marketing links in the same list
- hide the homepage entirely from logged-in users
- force users to log out to see marketing pages

==================================================
FINAL RESULT
==================================================

After implementing:

Users can:
- go from dashboard → homepage
- explore features while logged in
- revisit pricing
- share pages easily
- understand the product better

Your app will feel:
- cohesive
- professional
- complete

==================================================
FINAL TRUTH
==================================================

A real SaaS product is not:

“marketing site + separate app”

It is:

ONE CONNECTED SYSTEM

This change is small in code, but large in impact.
