# Quote Comparison Summary Feature

## Overview
This feature generates AI-powered comparison summaries when an issue has multiple contractor quotes, helping users make faster, more informed decisions. The summary is generated by Claude Haiku and highlights the most cost-effective option, best availability, and a clear recommendation.

## Components Created

### 1. API Endpoint: `app/api/issues/[id]/quote-summary/route.ts`

**Purpose**: Server-side endpoint that generates the AI comparison summary.

**Functionality**:
- Authenticates the user and validates issue ownership
- Fetches all dispatches and contractor responses for the issue
- Filters responses that have pricing information (flatEstimate, estimateLow, estimateHigh)
- Extracts key data from each quote:
  - Contractor name and company
  - Pricing (formatted as flat, range, or starting price)
  - Availability (text or formatted date)
  - Response time (minutes from dispatch to reply)
  - Additional notes
- Calls Claude Haiku to generate a concise comparison (2-4 sentences)
- Returns the summary or appropriate error message

**Request/Response**:
```
GET /api/issues/{id}/quote-summary
Authorization: Required (user must own the issue's property)

Response (200):
{
  "summary": "Smith Plumbing is the most cost-effective at $450 with same-day availability. Johnson Plumbing quoted $600 but can start tomorrow. Recommend Smith Plumbing for quick, affordable service."
}

Response (200, <2 quotes):
{
  "summary": null,
  "message": "Not enough quotes to compare"
}

Response (404):
{
  "error": "Issue not found."
}

Response (500):
{
  "error": "Error message"
}
```

**Technical Details**:
- Uses `getAnthropicClient()` from `lib/ai/client.ts`
- Model: `claude-haiku-4-5-20251001`
- Max tokens: 300 (sufficient for concise summary)
- Temperature: 0 (consistent output)
- Proper error handling and user authorization checks

### 2. Client Component: `app/(app)/issues/[id]/quote-summary.tsx`

**Purpose**: React component that displays the AI summary on the issue detail page.

**Props**:
- `issueId: string` - The issue ID to fetch the summary for
- `quoteCount: number` - Number of quotes (used to determine if component should render)

**Behavior**:
- Returns `null` if `quoteCount < 2` (only renders with 2+ quotes)
- Fetches summary on mount via GET `/api/issues/{id}/quote-summary`
- Manages three states:
  1. **Loading**: Shows animated skeleton placeholder
  2. **Error**: Shows error message with "Try again" button
  3. **Success**: Shows summary in a styled blue card with refresh capability

**UI States**:
```
Loading State:
- Card with "✨ AI Quote Comparison" header
- h-20 animated skeleton pulse

Error State:
- Card with error message
- "Try again" button to re-fetch

Success State:
- Blue-tinted card (border-blue-200, bg-blue-50/40)
- "✨ AI Quote Comparison" header with "Refresh" button
- Summary text in readable size and weight
- Professional spacing and typography
```

**Styling**:
- Blue accent color: `border-blue-200, bg-blue-50/40`
- Icons: Sparkle (✨) to indicate AI feature
- Responsive and mobile-friendly
- Consistent with existing component styling

## Integration

### Placement in Issue Detail Page

The QuoteSummary component is placed in `app/(app)/issues/[id]/page.tsx`:

**Location**: After "Contractor Contacts" section, before "Quote Responses" section

**Rendering Logic**:
```tsx
{/* AI Quote Comparison Summary */}
{allResponses.length >= 2 && issue.status === 'quotes_received' && (
  <QuoteSummary issueId={issue.id} quoteCount={allResponses.length} />
)}
```

**Conditions**:
- Only renders when 2 or more quotes are present
- Only renders when issue status is "quotes_received"

**Page Position**:
1. Header section
2. Status banners
3. Job lifecycle panel
4. AI Classification section
5. Description section
6. Photos section
7. Attachments section
8. Stale dispatch nudges
9. Contractor Contacts section
10. **AI Quote Comparison Summary** ← NEW
11. Quote Responses section (detailed quote cards)
12. Manual quote + actions
13. Activity timeline

## User Experience Flow

1. **Issue Created**: User reports a maintenance issue
2. **Dispatched**: Issue is sent to selected contractors
3. **Quotes Received**: Contractors submit quotes (status = "quotes_received")
4. **AI Summary Generated**: When 2+ quotes arrive, QuoteSummary fetches and displays
5. **Quick Decision**: User sees AI-generated comparison before diving into details
6. **Detailed Review**: User reviews full quote cards below
7. **Selection**: User selects a contractor and confirms

## Data Model

The feature leverages the existing Prisma schema:

**Relevant Tables**:
- `Issue` - Main issue record
- `Dispatch` - Records of contractor outreach
- `ContractorResponse` - Quote responses with pricing
- `Contractor` - Contractor information

**Key Fields Used**:
- `ContractorResponse.flatEstimate` - Single price quote
- `ContractorResponse.estimateLow` - Price range low
- `ContractorResponse.estimateHigh` - Price range high
- `ContractorResponse.availabilityText` - Availability description
- `ContractorResponse.availabilityDate` - Available date
- `ContractorResponse.notes` - Additional notes
- `ContractorResponse.receivedAt` - When response was received
- `Dispatch.sentAt` - When dispatch was sent (for response time)
- `Contractor.name` - Contractor name
- `Contractor.companyName` - Company name

## API Prompt Design

The prompt sent to Claude Haiku is designed to generate concise, actionable comparisons:

```
You are analyzing contractor quotes for a home maintenance issue. Generate a brief, actionable comparison summary (2-4 sentences max) that helps the homeowner make a decision.

Here are the quotes received:
Contractor 1: [name]
- Price: [formatted price]
- Availability: [availability]
- Response time: [minutes]
- Notes: [notes if any]

Contractor 2: [name]
...

Provide a concise summary that tells the homeowner:
1. Who is the most cost-effective option
2. Who has the best availability/fastest response
3. A clear recommendation with reasoning

Keep it conversational and practical. Focus on helping them choose.
```
