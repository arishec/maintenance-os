SESSION 3: FIXES + APP RUNNING (March 26, 2026)
========================================

Fixed 2 critical issues that were blocking the app:

1. DATABASE_URL hostname was wrong: aws-0 → aws-1
   - Old: aws-0-us-east-1.pooler.supabase.com (wrong, caused "FATAL: Tenant or user not found")
   - New: aws-1-us-east-1.pooler.supabase.com (correct, from Supabase Connect panel)
   - Both DATABASE_URL and DIRECT_URL updated

2. Redirect loop on dashboard (sign-in ↔ dashboard infinite loop):
   - Cause: dashboard used prisma.user.findUnique() which returned null for new Clerk users
     (no User row in DB yet), then redirected to /sign-in, which redirected back (already signed in)
   - Fix: replaced manual auth + findUnique pattern with requireDbUser() from lib/auth.ts
     which does an upsert (creates user on first visit)
   - Applied same fix to all 6 protected pages: dashboard, properties, issues, issues/[id],
     contractors, history

- Clerk auth: middleware.ts, sign-in/sign-up pages, ClerkProvider placement
- SDK lazy init: twilio.ts, resend.ts, ai/client.ts (avoid crash with placeholder keys)
- env.ts: split required vs optional keys
- issues/new/page.tsx: data structure mismatches

3. Smart quotes in contractors/page.tsx:
   - Agent used curly quotes (UTF-8 e2 80 98/99) instead of straight quotes (27)
   - Caused "Unexpected character '''" build error
   - Fixed with sed replacement

4. Settings page 404:
   - Sidebar linked to /settings but no page existed
