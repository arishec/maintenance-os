=== SMART CONTRACTOR RECOMMENDATIONS ON DISPATCH PAGE ===

Added intelligent contractor fit scoring and recommendations to the dispatch page.

FILE MODIFIED:
1. app/(app)/issues/[id]/dispatch/page.tsx

FEATURES ADDED:

1. computeScore() function (lines 95-135)
   - Calculates fit score (0-100) for each contractor based on:
     * Trade match to issue category: 40 pts (direct match), 20 pts (related like handyman)
     * Response rate: up to 25 pts (25 * responseRate ratio)
     * Completed jobs: up to 20 pts (min(completedJobs * 4, 20))
     * Response speed: up to 15 pts (faster = more points, tiered by hours)
   - Preferred contractors get +10 bonus applied in sort logic

2. getRecommendationText() function (lines 137-157)
   - Generates human-readable recommendation text for top contractors
   - Shows response time estimate (e.g., "Responds in ~2 hrs")
   - Shows completed jobs count
   - Format: "Responds in ~2 hrs · 3 jobs completed"
   - Returns null if insufficient stats data

3. Updated contractor sorting (lines 220-224)
   - Changed from simple isPreferred float to score-based ranking
   - Sorts all contractors by computed fit score in descending order
   - Applies +10 bonus to preferred contractors before sorting
   - Maintains flexibility with trade filtering and "Show all" toggle

4. Enhanced contractor card rendering (lines 423-491)
   - Added score indicator dot next to contractor name:
     * Green dot: score >= 70 (fit score visible in tooltip)
     * Yellow dot: score 40-69
     * Gray dot: score < 40
   - Added green recommendation badge for top contractor:
     * Shows if score > 70 AND is first in list
     * Includes star icon and recommendation text
     * Uses green-50 background with green-200 border
   - Maintains all existing features (checkboxes, contact info, channel selection)

SCORING BREAKDOWN (max 100 points):
- Trade relevance: 40 (primary) / 20 (secondary)
- Response rate: 25 (based on % of replies)
- Completed jobs: 20 (min of completedJobs*4 or 20)
- Response speed: 15 (tiered: <1hr=15, <2hr=13, <4hr=11, <8hr=9, <24hr=6, else=2)
- Preferred bonus: +10 (applied during sort)

IMPLEMENTATION NOTES:
- Uses existing stats data (totalDispatches, replied, responseRate, completedJobs, avgResponseMs)
- No API changes required; works with existing GET /api/contractors?stats=1
- Recommendation badge only shows for single top-ranked contractor (idx === 0 && score > 70)
- Score calculation integrated into existing tradeMap logic
- Color-coded dots provide visual quick-scan of contractor quality
- All changes are surgical edits to maintain code clarity
